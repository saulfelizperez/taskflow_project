# Backend API - Herramientas esenciales en desarrollo

-Este documento presenta una explicación reformulada de varias herramientas clave utilizadas en el desarrollo de APIs backend modernas: Axios, Postman, Sentry y Swagger. Se describe su función y la razón de su uso en la programacion.

---

```bash
## 1. Axios

**¿Qué es?**
-Axios es una librería basada en JavaScript que permite realizar solicitudes HTTP desde aplicaciones tanto en el navegador como en entornos Node.js.

**¿Para qué sirve?**
-Se emplea principalmente para establecer comunicación entre el cliente (frontend) y el servidor (backend), facilitando el intercambio de información con una API.

**¿Por qué se utiliza?**

- Simplifica el manejo de peticiones HTTP en comparación con alternativas nativas.
- Trabaja automáticamente con datos en formato JSON.
- Permite gestionar errores de forma más estructurada.
- Ofrece funciones avanzadas como interceptores de solicitudes y respuestas.

---

## 2. Postman

**¿Qué es?**

-Postman es una plataforma que permite probar y depurar APIs mediante una interfaz gráfica sin necesidad de desarrollar código adicional.

**¿Para qué sirve?**

-Se utiliza para enviar solicitudes a diferentes endpoints del backend y verificar su comportamiento y respuestas.

**¿Por qué se utiliza?**

- Facilita la comprobación rápida de endpoints.
- Ayuda a detectar fallos en la API durante el desarrollo.
- Permite organizar pruebas en colecciones reutilizables.
- Mejora la comprensión del funcionamiento de la API.

---

## 3. Sentry

**¿Qué es?**

- Sentry es una herramienta de seguimiento y control de errores en aplicaciones web y servidores.

**¿Para qué sirve?**

- Se utiliza para identificar fallos en tiempo real, recopilando información detallada sobre los errores que ocurren en producción o desarrollo.

**¿Por qué se utiliza?**

- Detecta automáticamente errores sin intervenció
```
